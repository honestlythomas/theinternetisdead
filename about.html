About

A map of the site, the shell, and the strange collection living inside it.

+--------------------------------------------------+ | THE INTERNET IS DEAD. THE PAGE STILL LOADS. | +--------------------------------------------------+

theinternetisdead.org is a personal web art archive by Thomas Harrison / Honestly Thomas / Tektite. It is part retro terminal, part experimental desktop, part game shelf, part media dump, and part hand-built proof that a website can still feel like a place.

The site runs as a static shell: the top navigation, starfield, command-like URL display, and sidebar stay around while the main iframe loads games, experiments, tools, videos, archives, and strange little utilities.

What Exists Here

The site is organized around a few big zones. Some are polished enough to use. Some are sketches. Some are artifacts from experiments that were worth keeping because they still have a pulse.

Games

The games section includes browser games such as Shooter Game and Tektite 3D Game. Expect arcade controls, canvas effects, audio, level assets, controller experiments, and deliberately loud old-web energy.

Experiments

The experiments area contains visual and interactive tests: canvas backgrounds, blotter-style visuals, chat sandbox systems, mandalas, photo spheres, sound tools, stacks, camera experiments, and other pages built to see what the browser can be pushed into doing.

Tools

The tools section is a grab bag of usable browser utilities: bookmark management, camera studio work, Dead TV, karaoke, multimedia editing, music playback, PDF viewing, and related interface experiments.

Videos And Feeds

The home feed and video routes surface archived video metadata, thumbnails, channel-style browsing, YouTube-related material, and large generated indexes that turn the site into a navigable media wall.

Downloads And Extensions

The downloads area includes extension-suite work and packaged site artifacts. It is where browser extension experiments, stored themes, icons, readmes, and downloadable builds collect.

Site Infrastructure

Behind the scenes are JSON indexes, thumbnail maps, build scripts, page helpers, nav overlays, chat command code, generated site maps, and enough glue to make a static site behave like a small operating system.

Notable Pieces

  • The iframe shell: a persistent frame around the site with sidebar navigation, URL state, a clock, and a starfield canvas behind the loaded page.
  • The thumbnail grid: the front-door browsing surface for many site entries, media items, and generated link previews.
  • Canvas backgrounds: standalone visual backgrounds including the starfield used by the shell.
  • Chat sandbox: a local command/interface experiment that can overlay parts of the site.
  • Dead TV and media tools: pages that treat media browsing as an interface playground rather than a plain list.
  • Repo-generated indexes: JSON and sitemap data that expose the shape of the whole project.

How To Read It

This is not a normal portfolio landing page. It is closer to a living archive: a site where tools, games, half-finished interfaces, archived media, old jokes, build outputs, and working experiments are allowed to coexist.

The useful parts are real. The rough parts are also real. The point is not to sand the site into a generic product surface, but to keep the weirdness browseable, playable, and recoverable.
